Assignment 1: Choropleth Map

Software Used: QGIS, VSCode, GIMP

Course Context & Objective

In this first part of the course, we were shown the basic elements of geographic data visualization. We covered different data types and the different ways to put values on a map. We also discussed different projections and the distortions associated with each. Finally, we went over different color combinations and saw how to choose the right colors for our map. In this assignment, we were asked to download an SDG indicator dataset from UNStats and make a choropleth map of the world showing how well each country is doing with this specific indicator.

Technical Skills Learned

  • Managing attribute tables and performing table joins in QGIS.
  • Applying mapping design principles (color, legends, titles)
  • Turning a .png map into an interactive web map.
  • Making the web map accessible to people with colorblindness.

Results & Analysis

My final map (shown below) shows annual forest area change rates in every country. Every country is given a color. Red indicates net forest losses, and green indicates net forest gains. More saturated colors indicate more extreme values. Hovering over a country will show the 2015 numerical data. Clicking on a country will show the 2000 data, 2015 data, and historical trend. Clicking on the "High Contrast Mode" button will change the colors to a palette that is more easily readable by people with colorblindness. I tested this palette for deuteranopia, protanopia, tritanopia and grayscale using the Windows built-in color filters. The original colors were unreadable for all the conditions except tritanopia. High contrast mode makes the map easily readable with every condition.

I used AI to help me with certain steps of this assignment. I used it to help me with the tabular joins in QGIS. I also asked it to give me instructions on how to change certain settings in the QGIS export layout window. It helped me convert my geoJSON file into a JS file by using a python script, and it helped me troubleshoot the button to switch the color palette, which I had difficulty making. At a few points during the website creation, I also asked it to make my code more neat because I would get lost with my indentations and variable names.

After receiving some in-class feedback on my QGIS map, I made a few changes to this interactive map. I slightly refined the colors of the 'normal' map to make them clearer for colorblind people. There was already a high-contrast version but I wanted to make the normal map as accessible as possible as well. I also slightly modified what the borders look like when they're highlighted, to avoid overlap between country borders.
